DAT 1001 Project 1 - Collecting and Analyzing Primary Data from a Remote Sensor

Introduction

In this project, you will collect primary data from a remote environmental sensor located in the Denver area. The sensor is a BME280 environmental sensor, connected to a Raspberry Pi which you can access through the internet. The sensor records temperature, air pressure, and humidity. You will configure the sensor to collect data which you will download and analyze, comparing your collected data to secondary data available on the internet.

An overview of the steps in this project is provided below, followed by more detailed step-by-step instructions. This project is worth 30% of your final grade. Please read the detailed grading rubric on D2L before submitting your work. Please ask for help wherever you need it.

What you need to submit

For this project you should submit a single tar file containing your whole project1 folder from your Jupyter Hub account. Instructions are provided on how to do this. The file should contain a single Jupyter notebook with your completed answers to all the tasks, along with images of screenshots where required, and your data files to the assignment link in D2L. Your notebook should include all the code, markdown, images and commentary as needed. Please double check that your notebook includes all these when you upload it to D2L.

Task 3 Detailed Instructions

In this task, you will create a bash script and use the built-in Linux scheduler cron to record data from the BME280 sensor at regular intervals.

Firstly, log into your account and activate the virtual environment as before:

source venv/bin/activate

Now test the following Python script that records data from the sensor:

If this runs correctly, you should be able to see the recorded data in a CSV file called data.csv. To check the file is there, type the following and hit enter:

cat data.csv

You should get something like this:

The cat command prints out the contents of any text file, which includes CSV files. You should be able to see a line of data in the file. It’s also a good idea to use cat with the bme280_save_data.py file so you can see the Python code.

Now you need to set up a bash shell script to run this Python code. We will use the nano text editor to create and save the script. To begin with, type

nano bme280_cronjob.sh

This will open the text editor. You will need to type the following three lines into the editor:

#!/bin/bash
source venv/bin/activate
python bme280_save_data.py
deactivate

Once you’ve done this, hit CTRL-X to exit, type Y to save and hit enter to save with the current filename.

Before you can run the script, you will need to make this script executable. To do this, type the following and hit enter:

chmod u+x bme280_cronjob.sh

Now you can test the script. Type the following:

./bme280_cronjob.sh

Then check the data.csv file by typing:

cat data.csv

You should see a new line of data added to your CSV file.

The final step is to set up a cron job, using the built-in Linux scheduler called cron.

Open crontab with the following command:

crontab -e

(You may be prompted to choose and editor, in which case use nano.)
